Guide

Everything you need to get a figure that survives peer review, plus a short primer for anyone who has never written a SMILES string.

Drawing one structure

  1. Enter the compound. Type a name and press Enter to resolve it through PubChem, or paste a SMILES string directly. SMILES is faster and works offline once the page has loaded.
  2. Check the depiction. The strip under the structure reports molecular weight, heavy-atom count, and ring count. If those numbers don't match what you expect, the input was probably not the molecule you intended.
  3. Choose a format. SVG for anything going into a manuscript. PNG when a submission system rejects vector files.
  4. Download. The file is written straight from your browser to your machine.

SVG or PNG

SVG stores the drawing as instructions rather than pixels, so it stays perfectly sharp at any size and typically weighs a few kilobytes. It is the right default for journals, theses, and LaTeX.

PNG is pixels at a fixed size. Use it when something downstream cannot accept vector files — some submission portals, some slide templates, most messaging apps.

Backgrounds

Downloads are white by default, which is what journals and slide decks expect and what avoids the classic failure of black-on-black when a file is dropped onto a dark theme.

Switch on Transparent background when you're compositing the structure over a coloured panel or a photograph. Two cautions: some manuscript systems flatten alpha to black rather than white, and Microsoft Word has historically mishandled transparent PNGs when printing. If a figure must simply work everywhere, leave the setting off.

Before submitting: open the downloaded file on its own, at full size. Depictions that look fine in a small preview often reveal overlapping labels or clipped substituents when viewed at print scale.

Building a panel figure

The panel builder composes several structures into one image, which is how most medicinal chemistry figures are actually laid out. One compound per line, in either form:

Aspirin | CC(=O)OC1=CC=CC=C1C(=O)O
Caffeine | CN1C=NC2=C1C(=O)N(C)C(=O)N2C
Warfarin

A line with a label and a SMILES string draws immediately. A line with just a name is resolved through PubChem first, so it needs a connection and takes a moment longer. Column count controls the grid shape; the exported file is a single figure, not a zip of separate images.

A short SMILES primer

SMILES writes a structure as a line of text. The essentials cover most drug-like molecules:

  • Atoms are element symbols. Carbon chains are just CCC; hydrogens are implied.
  • Bonds are implicit for single, = for double, # for triple.
  • Branches sit in parentheses: CC(C)C is isobutane.
  • Rings open and close with a matching digit: C1CCCCC1 is cyclohexane.
  • Aromatic rings use lowercase: c1ccccc1 is benzene.
  • Charges and isotopes go in brackets: [NH4+], [13C].
  • Stereochemistry uses / and \ for double bonds, @ and @@ for chiral centres.

If you don't want to write one by hand, look the compound up by name here, or copy the canonical SMILES from its PubChem page.

When something doesn't work

What you seeWhat it means
SMILES could not be parsedUsually an unclosed ring digit, an unbalanced bracket, or a capital C where an aromatic lowercase c belongs.
PubChem has no compound by that nameTry the systematic or generic name rather than a brand name, check the spelling, or paste SMILES instead.
The chemistry engine could not loadThe page fetches RDKit from a CDN on first load. Reload with a working connection; some institutional firewalls block CDN traffic.
Name lookup is slow or fails in bulkPubChem asks for no more than about five requests per second. Large panels of names take a moment; supplying SMILES avoids the limit entirely.

Privacy

Structures are rendered by WebAssembly inside your browser. Nothing you type or draw is sent to a server, and nothing is stored. The one exception is name lookup, which by necessity sends the name you typed to PubChem — if a structure is confidential, paste it as SMILES rather than looking it up.

Doing this in Python instead

For batch work over CSV files, reproducible scripting, or automatic caption generation, use the MolPic Python package:

pip install git+https://github.com/akinwumiishola5000/MolPic.git
molpic generate "aspirin" -o aspirin.svg --no-h --name "Aspirin"
